CMPUT 415

This is the most intense course I have ever taken. From the start of the semester I was constantly learning, and I enjoyed it a lot. I gained experience writing a lot of C++ code. I also gained experience reviewing other people's code and working in a team using sprint planning. I gained an understanding of how compilers work, and implemented a front-end for a language called Gazprea to LLVM IR. The basic process is as follows:

  • Parse the code in a given language to generate the parse tree (or abstract syntax tree). Any syntax errors can be caught here
  • Define the scopes and symbols for the variables in the program. Any errors related to scoping can be caught here.
  • Define the type of each symbol defined in the scope tree. Any errors related to misusing types can be caught here.
  • Check for any semantic errors.
  • Go through the abstract syntax tree and do code emission. In our case we generated LLVM IR.
From there on, the LLVM IR can be converted into any assembly level backend. There are also compiler optimizations that can be performed here, but I only learnt about the theory behind it, we didn't implement it.
